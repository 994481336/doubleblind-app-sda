# 飞行员双盲测试/复飞专项评估分析平台需求报告

## 1. 产品定位

本应用是一个基于 Streamlit 的评估数据分析工具，面向运输航空公司飞行员双盲测试和复飞专项模拟机评估场景。它不负责组织双盲测试本身，而是读取评估结束后形成的 Excel 数据采集表，自动完成解析、算分、统计、可视化和报告导出。

第一阶段采用固定格式兼容口径：

- 主支持当前双盲测试表。
- 同时支持当前 `files/` 中的复飞专项检查表。
- 不继续维护旧版机型硬编码评分标准。
- 扣分项名称以 Excel 真实表头为准。

## 2. 设计过程

应用按四层设计：

- 格式识别：根据分值行、评估列名称和扣分列数量识别双盲测试表或复飞专项检查表。
- 数据解析：读取 Excel 多级表头，生成真实扣分项名称，并区分评分行、平均/合计行、备注行和结论行。
- 分析聚合：兼容 `检查员` 和 `评估员` 两类原始字段，计算人员最终得分、单位统计、技术等级统计和高频扣分项。
- 会话导出：在当前会话中生成 HTML、Word 和 CSV，不持久化上传文件或分析历史。

## 3. 双盲测试含义

在本项目中，"双盲测试"更接近评估组织方式，而不是软件内的匿名化功能。业务特征包括：

- 随机抽取不同机型、不同技术等级的飞行人员或机组参与评估。
- 在模拟机中执行预设复飞或飞行科目场景。
- 检查员或评估员根据统一评估单独立记录扣分行为。
- 评估结果主要用于宏观分析，例如训练质量、公司差异、趋势风险和共性短板。

## 4. 支持格式

当前固定支持两类 Excel 格式。

| 格式 | 分值行 | 评估列 | 预期扣分列 | 示例 |
| --- | ---: | --- | ---: | --- |
| 双盲测试表 | 第 3 行 | 检查员 | 93 | 东航、上航、山航、厦航样表 |
| 复飞专项检查表 | 第 5 行 | 评估员 | 149 | 江西航复飞专项检查数据采集表 |

解析器会自动识别：

- 分值行。
- `检查员` 或 `评估员` 列。
- 负分值扣分列。
- 合计行、平均分行、备注行、结论行。
- 多级表头中的科目、评分项目、扣分标准。

## 5. 核心功能

- 上传一个或多个 Excel 文件。
- 展示解析预览：文件名、文件格式、工作表、人员数、评分行数、平均/合计行数、扣分列数、警告。
- 生成人员平均得分、单位统计、技术等级统计、扣分明细。
- 按文件格式和单位筛选。
- 展示整体情况、单位和技术等级对比、科目/评分项目失分、高频扣分项、综合考评、风险指标、个人扣分明细。
- 单位平均分图表应显示当前筛选范围整体平均分基准线；解析预览、单位统计等汇总表应按行数自动展开，避免多文件上传时看起来只显示部分单位。
- 导出 HTML 报告、Word 报告、飞行员平均得分 CSV、原始扣分明细 CSV、风险指标 Excel。
- 在 public app 场景下只做会话内处理：可下载结果，但不保存上传文件或分析历史。

## 6. 数据模型

内部统一使用以下核心数据：

- `ParseSummary`：文件名、文件格式、工作表、分值行、人员数、评分行数、平均/合计行数、扣分列数、警告。
- `PilotScore`：人员 ID、姓名、所属单位、技术等级、文件格式、评分人数、最终得分、平均失分。
- `DeductionEvent`：人员 ID、姓名、所属单位、技术等级、评分人员、科目名称、评分项目、扣分标准、扣分值、原始列号、来源文件。

内部字段统一为 `评估员` 以便兼容解析；页面和导出按当前数据范围显示原始称谓：仅双盲测试表显示 `检查员`，仅复飞专项检查表显示 `评估员`，混合上传时显示 `评分人员`。

## 7. Public App 约束

- Public app 可以直接通过 URL 访问。
- 应用不持久化上传文件、分析结果或导出文件。
- 真实 Excel、PDF、Word 报告、密钥和人员数据不得提交到公开项目文件中。
- `.gitignore` 必须持续排除 `files/`、`.env`、`.streamlit/secrets.toml`、`.DS_Store`。
- 页面必须保留安全提示：请勿上传不允许外传的数据。
